# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Single-user, LAN-only telemetry tool for Forza Horizon (primarily FH6, with FH5 support). A Go server captures UDP Data Out, persists it as Parquet + DuckDB, serves REST + WebSocket, and embeds a TanStack Start SPA in the same binary. The frontend dashboard browses captured sessions/stints and renders live + historical analytics.

`CONTEXT.md` is the canonical domain glossary — **read it before naming or modelling anything new** (Session, Stint, Lap, Tick, Packet, Stint Type, Bookmark, Snapshot, Track Path, Comparison have precise definitions and "avoid" lists; planned-but-unbuilt concepts are marked as such).

## Commands

Toolchain: Go 1.26, pnpm + Node, `just` orchestrator (optional; manual fallbacks below).

```bash
# Dev — run both side by side
just dev                     # spawns server + client dev (Vite :3000)
# or manually:
cd server && go run ./cmd/forza-telemetry serve
cd client && pnpm dev

# Single-binary production build (chains gen-types → client → copy → server)
just build
# Produces server/bin/forza-telemetry with client embedded

# Tests
cd server && CGO_ENABLED=1 go test ./...                          # all packages
cd server && CGO_ENABLED=1 go test ./internal/storage/ -run TestX  # single test
cd client && pnpm vitest run                                       # unit tests (pure core: store selector, smoothers, MSDF layout, …)
cd client && pnpm lint                                             # oxlint (errors fail the gate; warnings don't)

# Lint + format
cd server && go vet ./...                  # also runs as part of CI gate
cd client && pnpm exec tsc --noEmit        # type check only (pnpm build also runs this)
cd client && pnpm format                   # oxfmt

# Regenerate TS Tick types from the canonical Go schema
cd server && go run ./cmd/gen-types > ../client/src/types/tick.generated.ts
```

**CGO is mandatory** for any `go build`/`go test` that touches `internal/storage` — DuckDB binds via cgo. Set `CGO_ENABLED=1` explicitly; do not skip.

## Architecture

```
Forza UDP → ingest.Listener → parser (FH5 Sled/Dash, FH6 Dash)
         → tick.Enrich (compute derived fields)
         → stream.Broker (in-memory ring + fan-out)
         ├→ api.ws (WebSocket; msgpack tick frames)
         └→ storage.Writer (per-stint segmented Parquet + DuckDB metadata; 5-min rotation bounds crash loss, ADR 0011)
                    │
                    ├→ session + stint detector (session: 1h gap / game-clock regression; stint: 10min gap / IsRaceOn flip / car change; discards sub-2s / <180-tick / idle / no-car stints)
                    └→ aggregator (per-stint/lap summary + 1Hz preview)
api.REST (sessions, stints, sub-resources, ticks) ← reads DuckDB + Parquet
api.web (SPA embed)                              ← serves TanStack client _shell.html
```

**Go server** (`server/`):
- `internal/tick/tick.go` — the canonical superset `Tick` struct. ALL downstream code reads this shape. Schema is **additive only** (ADR 0003).
- `internal/ingest/` — UDP listener + parser registry (size-keyed). Adding a new packet variant = register a parser; no other code changes.
- `internal/stream/broker.go` — non-blocking pub/sub; slow subscribers get dropped frames rather than backpressuring ingest.
- `internal/storage/` — `Store` owns DuckDB; `Writer` owns the per-stint lifecycle (parquet writer, detector state, aggregate trigger on close). Stints shorter than 2s are discarded (parquet deleted, row removed) per ADR 0006.
- `internal/api/` — REST + WS. `server.go` registers routes; per-resource files (sessions, stints, ticks, ws). Tick-series endpoint reads parquet via DuckDB `read_parquet()` with a 60-second window cap + column whitelist.
- `internal/web/embed.go` — `go:embed all:dist`. SPA shell fallback (`_shell.html`) for unknown paths.
- `cmd/gen-types/` — Go AST walker that emits `client/src/types/tick.generated.ts` from `tick.Tick`. Tag-driven (`msgpack:"..."` becomes JSON key).

**Client** (`client/`):
- TanStack Start with `spa.enabled: true` — emits `dist/client/_shell.html` for static serving. The Go embed handler routes all unknown paths to that shell.
- Path alias `~/*` → `client/src/*`.
- `routes/` — file-based; `__root.tsx` wraps everything in `AppShell` (HeroUI Sidebar + main).
- `utils/ws.ts` + `utils/live-store.ts` — WebSocket client + Zustand store for live tick frames.
- `types/tick.generated.ts` — generated by `gen-types`; do not hand-edit (regenerate from server).
- `DESIGN.md` — locked Glass theme reference (HeroUI Pro).

## Non-obvious gotchas (each bit us; preserved here so it doesn't bite again)

- **Glass theme dark mode ships `--background` dark but leaves `--foreground` light** — explicit overrides for all base dark tokens live in `client/src/styles/app.css` `.glass-dark` block. Do not remove. `--accent-foreground` is deliberately inverted (eclipse) so dark text reads on the near-white accent button.
- **DuckDB integer division returns DOUBLE** (`BIGINT / INTEGER`). Use `//` (integer division) when the result feeds into `PARTITION BY` or `CAST AS INTEGER`. The aggregator's preview-sample SQL bucket arithmetic learned this the hard way (`storage/aggregate.go`).
- **`parquet-go` cannot describe `[4]float32`** (fixed Go arrays). `storage/row.go` mirrors `tick.Tick` with `[]float32` per-wheel slices. When adding fields to `Tick`, mirror them in `parquetRow` + `toParquetRow`.
- **`@iconify/react` silently 404s missing icon names** — render as empty `<span>`. Use `lucide:*` (universal set). `gravity-ui:*` only ships a handful of icons despite docs implying it's the canonical set.
- **TanStack Start SPA build emits `_shell.html`, not `index.html`** — `server/internal/web/embed.go` fallback targets `_shell.html`. The `_shell.html` plus all dist/client/* are embedded into the Go binary via `just copy-client-dist`.
- **`gen-types` is one-way** — Go `Tick` → TS `TickFrame`. Hand-editing the generated file gets stomped. Add fields in `tick.go` first, then `parquetRow`, then regenerate.
- **`CurrentRaceTime > 0` is the freeroam-vs-race discriminator** — but it *classifies* stints at close, it never splits them (ADR 0013); splits come from gap / IsRaceOn / car only. See `data-needed.md` for confirmation status against real race captures.
- **DuckDB is single-writer.** Only one `forza-telemetry serve` process can hold the lock on `forza.duckdb` at a time. Starting a second instance fails with `Could not set lock on file ... Conflicting lock is held in <other-pid>`. Kill the prior process (`pgrep -f forza-telemetry | xargs kill`) before relaunching.
- **Vite 8 uses rolldown, which logs `INVALID_ANNOTATION` non-fatally** against HeroUI Pro's pre-minified `/*#__PURE__*/` comments. `vite.config.ts` filters them via `build.rollupOptions.onwarn`. Build still succeeds either way; the filter is to keep real warnings visible.
- **`react-resizable-panels@4` renamed its exports and changed size units** — it's `Group` (prop `orientation`, not `direction`) + `Separator`, not `PanelGroup`/`PanelResizeHandle`; `Panel` is unchanged. And **numeric** `defaultSize`/`minSize` are *pixels* — percentages must be strings (`"45%"`). A bare `defaultSize={45}` silently renders a 45px panel; `tsc` and the build stay green, only the browser shows it. See `components/LivePreview.tsx`.
- **MediaMTX's empty/built-in config has no catch-all path** — run with no config file it rejects all publishing with `path '<name>' is not configured`, which OBS surfaces as a misleading "could not access the specified channel or stream key". The repo ships a 2-line `mediamtx.yml` (`paths: all_others:`) that `just media` passes explicitly; all other settings (WebRTC :8889, no auth) stay default. See ADR 0010.

## Workflow conventions

- **Conventional Commits, granular.** One commit per logical concern; do not blanket `git add .`. The history in this repo follows `<type>(<scope>): <description>` with scopes like `server/storage`, `server/api`, `client`. Example: `feat(server/storage): add hot-spot detector` vs `fix(client): swap icons from gravity-ui to lucide`.
- **ADRs are locked.** If you find yourself contradicting one, stop and ask the user — don't drift the design.
- **Empirical unknowns belong in `docs/data-needed.md`.** When you guess at a Forza wire-format detail or a threshold, log the assumption there. When a real capture confirms or refutes it, remove or update the entry in the same change.
- **DESIGN.md is the locked design system** for the client. Use semantic HeroUI tokens (`bg-surface`, `text-foreground`, `text-muted`, `bg-accent-soft`, `shadow-surface`, `rounded-2xl`). Never inline hex/OKLCH in components.
- **Project memory lives in `~/.claude/projects/<project-hash>/memory/`** and points at `docs/data-needed.md`. Update both when new data resolves an unknown.
